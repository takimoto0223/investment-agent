## 【投資エージェント固有】

### プロジェクト目的
JP(kabu STATION API・現在未接続) / US(Alpaca paper) の2市場、
ScalpDay_JP/US + MomentSwing_JP/US の4ポッドで自動投資を行う。
取引コストが運用益を圧迫しないよう手数料を意識する。特に売買回数の多い ScalpDay はコスト効率に注意。
具体的なコスト基準はペーパー〜実弾のデータが貯まってから根拠を持って決める。

### アーキテクチャ原則
- v6組織：CIO → [Trader ← Critic(1:1)] / CXO(レポート専任)
- 全エージェントは agents/base.py の `BaseAgent` を継承
- 命名：エージェント=`{Strategy}_{Market}`、データクラス=`{Strategy}{Role}`
- 戦略識別子2値：`"scalpday"` | `"momentum_swing"`
- CIO集約：`MarketContext` 生成 + `allocate_budgets` を `CIOAgent` 1箇所に
- CXO は整形・送信専任：データは main.py が `CXOReportContext` に詰めて渡す

### データの素性（毎回の前提）
- USD/JPY : Frankfurter API（ECB当日値）← FRED DEXJPUS は実測7営業日遅延、使わない
- VIX/米10Y: FRED API (VIXCLS / DGS10)、T+1遅延だが方向感に十分
- US建玉  : Alpaca 実値。空なら「保有なし」表示。ダミー3銘柄（NVDA/MSFT/AAPL）は廃止済み
- JP建玉  : agents/cxo.py の `_DUMMY_JP` 固定ダミー。kabu API 接続後に実配線
- JP現金  : `allocate_budgets` の JP 元金は定数（kabu 接続後に実値へ）
- `allocate_budgets` には必ず `usd_jpy_rate` を渡す（固定値禁止）
- 全ライブデータ：24h JSON キャッシュ + ⚠暫定値フォールバック

### 命名の地雷（廃止・維持）
- "バリュー投資" / `ValueDecision` → 廃止。"スイング" / `SwingDecision` を使う
- `daytrade*` フィールドは Alpaca のドメイン用語として維持（改名しない）
- `ScalpDayCandidate` = Critic 審査前の候補、`SwingDecision` = 審査後の決定

### Skill に分離（詳細手順はここに書かない）
- 新しいデータソース追加（Frankfurter / FRED のパターン踏襲）
- verify_cxo_reports の使い方・テストパターン
- kabu API 配線手順（接続後に作成）

## 【プロジェクト共通の開発作法】
*このセクションは他プロジェクトに流用できる普遍的な原則。*

### 絶対に守る原則
- **実態と違う数字を出さない**: ダミー値・定数を実値として出力・表示しない
- **フォールバックは気づける形で**: 暫定値・推定値には ⚠ ラベルをつけ、利用者が判断できるようにする
- **スコープ外変更を勝手に入れない**: 指示された範囲を超える変更が必要と判断したら、
  入れる前に理由を添えて報告・確認する

### スクリプトの出力規約
- **非ASCII出力は文字化け対策を必須とする**: コンソール・ログに日本語など非ASCII文字を出力する
  スクリプトを新規作成・編集するときは、`import` 群の直後に必ず追加する:
  ```python
  sys.stdout.reconfigure(errors="backslashreplace")
  ```
  文字コードを特定値に固定しない理由: 化けの原因は出力側とコンソール側のコード不一致なので、
  特定エンコードへの固定より「表現できない文字をクラッシュや化けでなく `\uXXXX` 表記に倒す」安全策をとる
- **既存スクリプトへの後付け**: 触る機会があればその際に同じ対応を入れて揃えていく

### 開発ワークフロー
1. **棚卸し**: 着手前に影響範囲を調べ、確認してから実装に入る（いきなり着手しない）
2. **刻む**: 大きい変更は小さく分割し、各段階で動作を確認する
3. **実装**: 調査・確認済みの範囲を実装する
4. **テスト**: テスト緑 + サボタージュテストで回帰を防ぐ
5. **リファクタ**: 出力不変を verify で確認してからコミット
6. **記録**: backlog.md を完了・残タスクの唯一の置き場として更新する
